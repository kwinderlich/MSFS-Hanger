# MSFS 2024 Airport Overlay Input Steps

Preferred test input for one airport:

1. Airport source XML from the MSFS project, or a JSON export with equivalent runway, taxi, and parking data.
2. Airport identifier (ICAO / FAA / local ID).
3. Runway polygons and threshold numbers.
4. Taxiway points and taxiway paths.
5. Parking spots with size/type if available.
6. Start positions / spawn points if available.
7. Jetways / service stands if available.
8. One screenshot from the simulator showing the intended airport layout.

The current compiled scenery package is useful for package detection and metadata, but source XML or exported structured airport data is much better for clean vector overlays.
